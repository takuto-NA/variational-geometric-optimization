## 査読観点（モチベーション整合）

- 定義（座標自由）: \(d\mathcal F\)・\(d^2\mathcal F\)・停留条件が、内積や座標に依らずまず定義されているか
- 選択（表現）: 計量 \(G\)（Riesz 同一視）・散逸/保存（\(K,J\)）が「選択」であることが明確か
- 実装（可計算化）: 離散化・AD・線形作用（matvec/HVP）へ落とす最短経路が一貫しているか

## 章ごとの指摘（3〜5点）

### 第0章（序文）

- 良い点: 立ち位置（翻訳層）とスコープ宣言が明確で、読者の期待値調整ができている。
- 改善点: Part I の「章責務（各章1行目的）」が、`index.md` と二重化しない形で固定されるとより強い。
- 改善点: 「設計自由度（knobs）」が序文に出ていないため、以降の章で同じフレームが繰り返される“約束”が弱い。

### 第1章（変更前：中核定義）

- 課題: 一次・二次変分と計量の説明が第2章と重複しており、読者が「どこが定義の本丸か」を迷う。
- 改善案: ここは モチベーションの回収（問題設定＋設計自由度の提示）に責務を寄せると、Part I 全体の導線が締まる。
- 改善案: 数式は最小にし、`Space / Discretization / Geometry / Algorithm` の4箱を定義として固定する。

### 第2章（変更前：最小構成要素）

- 良い点: 一次変分→計量→二次変分→流れ/条件の並びは、定義→選択→目的の流れとして強い。
- 課題: 2.6（離散化）と 2.8（実装チェック）が入り、章タイトル「最小構成要素（理論）」の責務がブレる。
- 改善案: 2.6 と 2.8 は第8章へ移し、第2章は 停留構造の定義の唯一の置き場 に戻す。

### 第3章（統一方程式）

- 良い点: 反対称/対称分解と散逸不等式が1行証明で提示され、モチベーションの「構造の反復出現」を回収できている。
- 改善点: 第1章で “knobs” を固定した後に読むと強い章なので、章冒頭に「どのknobがここで固定されるか」を1〜2行で明示するとよい。

### 第4章（停留点）

- 良い点: 線形化と局所挙動（収束・発散・減衰振動）が、統一方程式の読み方として自然に繋がっている。
- 改善点: 「二次情報（地形）」と「流れ（\(K,J\)）」の責務分担は強いので、章末まとめで “knobs” に再投影（どれが選択か）すると学習効率が上がる。

### 第5章（手法マップ）

- 良い点: 流れ/反復/離散化の混線を3軸で整理しており、実務者向けの読み物として価値が高い。
- 課題: “knobs” フレームが章内で統一されていないため、表の読解が章ごとのローカル規約になりがち。
- 改善案: 各節冒頭の1行に「ここで主に触るknob」を明示（例：Algorithm と Discretization）。

### 第6章（制約）

- 良い点: 制約を「極小化」ではなく ラグランジアン停留構造 として統一しており、モチベーションに完全整合。
- 改善点: 実装章（第8章）への接続点（何を実装するか）を、章末サマリで1行だけ強めるとよい。

### 第7章（分野横断表）

- 良い点: 「同じに見せる」のではなく「違いを見える化する」と明示しており、安全で誠実。
- 改善点: 本来は第1章の“knobs”の実演なので、序盤に予告し、ここで回収する構造が望ましい。

### 第8章（実装への接続）

- 良い点: AD/HVP/線形作用/検証まで揃っており、実務者に刺さる。
- 課題: 第2章の離散化・実装チェックが残ると、ここで同じ話を再掲する必要が出て冗長化する。
- 改善案: 離散化（質量行列/剛性行列としての計量）と実装チェックは第8章へ集約し、ここを 実装の唯一の置き場 にする。

## リンク/章番号の移行方針（URLを壊さない）

章の責務を入れ替える際に最も危険なのは「URL（外部リンク）破壊」と「章参照の崩壊」である。本書では次を方針とする：

- URL（パス）は維持: `docs/chapters/chap01-core-definition.md` 等のファイル名/パスは原則固定し、表示タイトルとサイドバー順で新しい章責務を表現する。
- 表示文言は一括更新: `第X章` の表記、見出し、ロードマップ（`docs/index.md`・`chap00-preface.md`）を同期させる。
- 章内の重複は参照へ: 定義の「唯一の置き場」を決め、他章は「参照（リンク）」で回す（同じ定義を複製しない）。

移行チェックの最短手順：

1. サイドバー（`docs/.vitepress/config.mts`）と Home/Preface のロードマップを先に更新\n
2. 章本文を移設・統合し、旧タイトル文字列の残りを全文検索でゼロにする\n
3. `npm run docs:build` が通ること（リンク切れ・KaTeX警告の増減）を確認する
