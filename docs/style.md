## 数式（KaTeX）

- インライン: `$ ... $`
- ブロック: `$$ ... $$`
- 長い式は `\\begin{aligned} ... \\end{aligned}` を使う（`$$` の中）

## 記法の約束（最小）

- 対象空間: $\\mathcal M$
- 汎関数: $\\mathcal F: \\mathcal M\\to\\mathbb R$
- 計量（散逸）: $G$（対称正定値）
- 反対称構造（保存）: $J$（反対称）

## 応用例ページのテンプレ

応用例は原則として以下の順に書く：

- Problem
- Functional
- Geometry (G, J)
- Discretization
- Algorithm
- Notes

## 想定読者（読みやすさの基準）

本書は連続最適化に慣れた学部生を想定し、次は既知として省略することがあります：

- $\nabla f$ や $\nabla^2 f$ の基本的意味
- Newton 法の更新式
- KKT 条件の存在

その代わり、多様体／関数空間に拡張したときに「どこが同じでどこが変わるか」を明示する方針です。

## 文体（フォーマル規約）

- 節の役割を明確にするため、以下のラベルを一貫して用いる：
  - Definition: 用語・対象・仮定を定義する
  - Proposition: 本書で用いる主張を述べる（必要なら仮定を併記）
  - Remark: 読み方・直感・注意点（主張と混同しない）
  - Example: 具体例（計算例・既知モデルへの対応）
- “最適化”という語は、狭義（極小化）と広義（停留構造の数値的取扱い）を区別し、広義で用いる場合は明示する。
- “〜と考えられる/〜っぽい”などの曖昧表現は避け、仮定・定義・結論を分離して書く。
